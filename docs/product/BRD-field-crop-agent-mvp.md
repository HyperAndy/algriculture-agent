# AI 驱动的大田作物稳产减灾决策 Agent BRD

## 1. 文档目的

本文档用于说明“AI 驱动的大田作物稳产减灾决策 Agent”的商业背景、建设价值、目标用户、应用场景、收益预期与资金申请可行性，为项目立项、资金申报、产品评审和后续商业化提供依据。

## 2. 项目概述

本项目面向水稻、小麦、玉米、大豆等主要大田作物，建设一个集数据采集、智能诊断、风险预警、农事决策、任务闭环和档案追溯于一体的 AI Agent 系统。

系统通过整合气象、土壤墒情、作物生育期、田间观测、病虫草害描述、农事记录等数据，利用多 Agent 协作和长链推理机制，为农业经营主体、农技人员和管理部门提供地块级农事决策建议，提升大田作物稳产能力、灾害应对能力和绿色生产管理水平。

MVP 阶段采用“宣传演示大屏 + 管理后台”的双界面形态：宣传大屏用于资金申请、项目汇报和产品展示；管理后台用于地块管理、数据录入、Agent 分析、报告查看、农事任务和历史档案管理。宣传演示大屏最终采用政府/项目申报指挥中心风，包含中央田间风险态势地图、综合态势、作物分布、多 Agent 决策链、风险预警、农事任务队列和动态分析结果，并通过动画和交互增强项目展示说服力。

## 3. 商业背景

### 3.1 政策背景

粮食安全、智慧农业、农业防灾减灾、绿色种植和农业社会化服务是当前农业现代化建设的重要方向。水稻、小麦、玉米、大豆是保障粮食安全和重要农产品供给的核心作物，大田作物生产对稳产保供、节本增效和减灾防损具有重要意义。

随着物联网、遥感、人工智能和大模型技术的发展，农业生产正在从经验驱动向数据驱动、智能辅助决策转型。通过 AI Agent 将多源农业数据转化为可执行农事建议，具备较强的政策适配性和示范推广价值。

### 3.2 行业痛点

1. 大田面积大、地块分散，农技服务覆盖不足。
2. 苗情、墒情、病虫草害主要依赖人工巡田，发现和响应不及时。
3. 灌溉、施肥、用药决策依赖经验，存在过量投入、成本浪费和绿色生产压力。
4. 极端天气频发，干旱、暴雨、高温、低温、倒伏等风险缺少地块级预警和处置建议。
5. 农事记录分散，难以支撑补贴申报、保险理赔、绿色认证、项目验收和生产追溯。
6. 传统农业信息化系统多停留在数据展示层，缺少“分析-建议-任务-复盘”的闭环能力。

## 4. 项目目标

### 4.1 总体目标

建设一个可运行、可演示、可扩展的大田作物智能决策 MVP，验证 AI Agent 在大田作物稳产减灾、精准农事管理和农技服务提效中的应用价值。

### 4.2 阶段目标

MVP 阶段目标：

- 支持水稻、小麦、玉米、大豆四类作物。
- 支持示范地块管理和基础农情数据录入。
- 支持多 Agent 协作分析流程。
- 支持生成长链推理式农事决策报告。
- 支持自动生成农事任务并跟踪状态。
- 支持历史分析档案沉淀。
- 支持产品宣传大屏和实际使用后台。
- 支持无外部 AI Key 的 Mock Agent 演示模式。

二期目标：

- 接入真实天气 API、土壤传感器、虫情灯和遥感/无人机数据。
- 接入视觉模型识别病虫害和苗情。
- 接入真实大模型生成更自然的农事报告。
- 支持 GIS 地图、移动端巡田、PDF 报告导出和用户权限体系。

## 5. 目标用户

### 5.1 主要用户

- 农业合作社管理者
- 种植大户
- 农场经营主体
- 农技推广人员
- 农业园区/示范基地运营人员

### 5.2 次要用户

- 农业主管部门
- 农业项目评审人员
- 农业保险服务人员
- 农资服务商
- 农业数字化解决方案厂商

## 6. 核心应用场景

### 6.1 资金申请与项目演示

项目方通过宣传演示大屏展示大田作物风险态势、多 Agent 决策流程、典型报告、任务闭环和预期成效，提升项目表达清晰度和评审通过率。

该大屏应具备项目级指挥中心观感，而不是普通后台页面。核心展示形态为深色数字农业指挥舱：顶部状态栏展示天气、时间和系统状态，中央展示田间风险态势地图，左右展示综合态势、多 Agent 决策链和风险预警，底部展示土壤墒情趋势、农事任务队列和近期分析结果。

### 6.2 农技员日常诊断

农技员录入或查看地块数据，发起 Agent 分析，快速获得风险判断和农事建议，用于指导农户巡田、灌溉、施肥、防治病虫草害。

### 6.3 合作社统一管理

合作社可集中管理多个地块，查看高风险地块、待办农事任务和历史诊断记录，提高生产管理标准化水平。

### 6.4 灾害预警与应对

系统结合未来天气、作物生育期和土壤墒情，提前识别干旱、高温、暴雨、倒伏等风险，生成地块级应对建议。

### 6.5 农事档案与追溯

每次诊断、建议、执行任务和记录沉淀为农事档案，为项目验收、生产追溯、补贴申报、保险理赔和绿色认证提供基础资料。

## 7. 核心价值

### 7.1 对农业经营主体

- 降低巡田和诊断成本。
- 减少盲目施肥、过量用药和无效灌溉。
- 提升灾害和病虫害响应速度。
- 提高生产管理标准化程度。

### 7.2 对农技服务体系

- 将专家经验沉淀为可复用 Agent 流程。
- 提升农技服务覆盖半径。
- 降低重复咨询和人工初筛压力。
- 支持基层农技服务数字化升级。

### 7.3 对资金申请和项目建设

- 痛点清晰：稳产、减灾、节本、绿色、追溯。
- 技术路径清晰：多源数据 + 多 Agent 协作 + 长链推理。
- 成效可量化：响应时间、任务闭环率、风险识别率、投入品节约率。
- 可扩展性强：可逐步接入硬件、遥感、视觉模型和真实大模型。

## 8. 商业模式设想

MVP 阶段以项目制和示范基地建设为主。

后续可扩展商业模式：

- SaaS 订阅：按地块数量、作物类型或用户数量收费。
- 项目交付：为合作社、园区、农业企业定制智慧农业管理系统。
- 数据服务：提供风险预警、农事报告、生产档案和验收材料生成服务。
- 保险/金融合作：为农业保险、涉农金融提供风险评估和过程数据支撑。
- 农资服务合作：结合精准施肥、用药建议，提供合规农资服务入口。

## 9. MVP 成功指标

### 9.1 产品指标

- 支持 4 类作物：水稻、小麦、玉米、大豆。
- 至少内置 4 个示范地块。
- 完成一次完整 Agent 分析流程时间不超过 10 秒。
- 每次分析生成不少于 6 个 Agent 步骤结果。
- 每次分析生成不少于 3 条农事建议。
- 每次分析自动生成至少 1 条农事任务。

### 9.2 演示指标

- 3 分钟内可完整演示项目价值和业务闭环。
- 评审可在首页看清核心痛点、核心流程和预期成效。
- 管理后台可完成“录入数据-发起分析-查看报告-生成任务-沉淀档案”的完整流程。

### 9.3 业务预期指标

- 农技诊断响应从天级缩短至分钟级。
- 减少盲目施肥和不规范用药。
- 提升高风险地块发现和处置效率。
- 提升农事记录完整度，为项目验收和补贴申报提供支撑。

## 10. 主要风险与应对

### 10.1 数据真实性不足

MVP 使用模拟数据和人工录入数据，可能无法完全代表真实生产环境。

应对：明确 MVP 目标是验证流程闭环；二期接入真实天气、传感器和遥感数据。

### 10.2 AI 决策专业性不足

Mock Agent 规则有限，不能替代农业专家。

应对：将系统定位为辅助决策；引入农技专家规则库和可配置阈值；后续接入专业模型。

### 10.3 过度承诺自动化能力

如果宣传为全自动控制系统，会增加落地风险。

应对：MVP 定位为“农事决策辅助 + 任务闭环”，不承诺自动控制灌溉或农机设备。

### 10.4 一期范围过大

如果同时做 GIS、硬件、视觉识别、移动端，会拖慢 MVP。

应对：一期只做 Web、手动/模拟数据、Mock Agent 和核心闭环。

## 11. 结论

本项目兼具政策契合度、农业场景刚需、AI 技术展示度和工程可落地性。MVP 阶段通过宣传演示大屏和管理后台证明业务闭环，后续可逐步接入真实数据、视觉模型、物联网设备和大模型能力，形成可推广的大田作物智能稳产减灾解决方案。
