# 大田作物稳产减灾 Agent 演示脚本与评审话术

## 1. 文档目的

本文档用于项目路演、资金申请答辩、产品演示和内部验收，帮助演示者在有限时间内讲清楚项目痛点、技术逻辑、产品闭环和预期价值。

## 2. 30 秒电梯介绍

我们做的是一个面向水稻、小麦、玉米、大豆的大田作物稳产减灾决策 Agent。它不是普通聊天助手，而是围绕地块、作物生育期、天气、土壤、苗情、病虫草害和农事记录构建的多 Agent 决策系统。系统可以把分散的农情数据转化为风险预警、农事建议、执行任务和可追溯档案，帮助农技员和合作社把经验式管理升级为 AI 辅助的精准农事管理。

## 3. 3 分钟演示脚本

### 0:00-0:30 打开宣传大屏

打开 `/demo`。

话术：

“这是项目的宣传演示大屏，采用政府/项目申报指挥中心风格。顶部展示天气、湿度、风向、时间和系统状态，中央是田间风险态势地图，左侧是综合态势和作物结构，右侧是多 Agent 决策链和风险预警。我们聚焦四类主要大田作物：水稻、小麦、玉米、大豆，解决农技覆盖不足、风险发现晚、水肥用药粗放、农事过程缺少完整追溯的问题。”

### 0:30-1:00 展示核心流程

指向右侧多 Agent 决策链和中央地图动效。

话术：

“系统采用多 Agent 协作，不是单次问答。右侧节点会按顺序点亮，箭头流光体现决策链正在运行。田间感知 Agent 负责整理数据，作物诊断 Agent 判断生育期和长势，病虫草害预警 Agent 判断生物风险，水肥决策 Agent 生成灌溉和追肥建议，灾害应对 Agent 识别干旱、高温、暴雨等风险，农事档案 Agent 负责沉淀追溯记录。”

### 1:00-1:20 展示地图联动和预警

点击高风险预警或中央高风险地块。

话术：

“中央地图不是静态背景。高风险地块有脉冲提醒，传感器点位有信号波纹。点击预警可以联动高亮对应地块，点击图层按钮可以切换气象、土壤、病虫、灾害等展示状态。”

### 1:20-1:50 进入管理后台

打开 `/console`。

话术：

“除了大屏展示，我们也做了实际可用的管理后台。这里可以查看地块总数、高风险地块、待执行任务和近期分析记录，适合农技员和合作社日常使用。”

### 1:50-2:30 演示玉米地块分析

进入“东北玉米示范田”，录入高温少雨、墒情偏低、叶片偏淡、虫害描述等数据，点击分析。

话术：

“现在我们模拟一个玉米拔节期场景：未来 5 天高温少雨，土壤墒情偏低，叶色偏淡，田间发现少量虫咬叶片。系统会先判断作物和生育期，再融合天气、土壤、苗情和病虫害信息，经过多个 Agent 逐步推理。”

### 2:30-2:50 展示报告和任务

打开报告详情。

话术：

“报告不是只给一句结论，而是包含风险等级、判断依据、长链推理、农事建议和后续观察指标。比如这里会提示干旱风险、建议补灌、避开高温追肥，并自动生成巡田和复查任务。”

### 2:50-3:00 展示档案

打开 `/console/archive`。

话术：

“每次分析都会进入农事档案，可用于生产追溯、项目验收、补贴申报和保险理赔的数据支撑。这就是从数据输入到智能诊断、任务执行、档案沉淀的完整闭环。”

## 4. 8 分钟详细演示脚本

1. 介绍政策背景：粮食安全、智慧农业、稳产减灾、绿色种植。
2. 介绍行业痛点：农技覆盖不足、风险发现晚、水肥用药粗放、档案分散。
3. 展示 `/demo`：项目对象、核心指标、Agent 流程、预期成效。
4. 进入 `/console`：说明它不是纯展示页，而是业务后台。
5. 查看四类示范地块：水稻、小麦、玉米、大豆。
6. 选择玉米示范田：说明作物、生育期、面积和风险。
7. 录入观测数据：天气、土壤、苗情、病虫草害、农事记录。
8. 发起 Agent 分析：讲解六 Agent 职责。
9. 查看报告：讲解长链推理和建议依据。
10. 查看任务：说明系统推动执行。
11. 查看档案：说明追溯和验收价值。
12. 说明二期扩展：天气 API、IoT、遥感、视觉模型、真实 LLM。

## 5. 评审常见问题回答

### 5.1 这是不是只是套了一个 ChatGPT？

不是。系统不是开放式聊天，而是围绕地块对象、作物生育期、多源农情数据和农事任务闭环构建的 Agent 流程。MVP 阶段先用规则型 Mock Agent 保证稳定演示，后续可替换为真实 LLM、视觉识别模型和农技专家规则库。

### 5.2 没有真实传感器，项目是否可落地？

MVP 的目标是验证业务闭环：数据输入、Agent 分析、报告生成、任务执行和档案沉淀。真实传感器、天气 API、遥感和无人机数据属于二期接入，系统架构已经预留接口。

### 5.3 AI 给出的农事建议是否可靠？

项目定位是农技辅助决策，不替代专家现场判断。MVP 使用明确的风险规则保证输出可解释，未来会结合专家规则库、真实数据和模型评估持续优化。

### 5.4 项目能带来什么量化价值？

预期可以把农技初步诊断从天级缩短到分钟级，减少盲目施肥、盲目用药和无效灌溉，提高高风险地块发现效率，同时沉淀可追溯农事档案，支撑项目验收、补贴申报和保险理赔。

### 5.5 为什么选择水稻、小麦、玉米、大豆？

这四类作物覆盖主要粮食和重要油料作物，关系粮食安全和稳产保供，政策价值和示范推广价值更强，也更适合做资金申请项目。

## 6. 项目亮点话术

- “一套系统，两类场景：既能用于项目展示，也能用于农技员日常管理。”
- “不是单点 AI 问答，而是多 Agent 协作的农事决策闭环。”
- “先用 Mock Agent 保证 MVP 稳定运行，后续平滑替换为真实大模型和视觉模型。”
- “从风险识别到任务生成再到档案沉淀，解决的是农业生产管理的完整链条。”
- “项目聚焦大田作物稳产减灾，符合智慧农业、粮食安全和绿色生产方向。”
