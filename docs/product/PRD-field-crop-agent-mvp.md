# AI 驱动的大田作物稳产减灾决策 Agent PRD

## 1. 产品概述

### 1.1 产品名称

AI 驱动的大田作物稳产减灾决策 Agent

### 1.2 产品定位

面向水稻、小麦、玉米、大豆等大田作物的智能农事决策系统，通过多 Agent 协作和长链推理，将地块数据转化为可执行的风险预警、农事建议、任务计划和生产档案。

### 1.3 产品形态

MVP 包含两个界面：

1. 宣传演示大屏：用于项目展示、资金申请和产品介绍。最终视觉必须采用政府/项目申报指挥中心风，详见 `docs/product/UI-VISUAL-SPEC-field-crop-agent-command-center.md`。
2. 管理后台：用于农技员、合作社和农场管理者进行日常地块管理和农事决策。

### 1.4 产品目标

- 让评审和项目方快速理解项目价值。
- 让用户完成真实可用的农技决策闭环。
- 验证多 Agent 在农业种植业中的落地路径。
- 为后续接入真实数据和大模型能力保留扩展接口。

## 2. 用户角色

### 2.1 项目评审/参观者

关注项目是否解决真实问题、技术路线是否清楚、成效是否可量化。

主要使用 `/demo`。

### 2.2 农技员

关注地块状态、风险诊断、农事建议和任务安排。

主要使用 `/console`、地块页、数据录入页、Agent 分析页、报告页。

### 2.3 合作社/农场管理者

关注多个地块的风险态势、待办任务、历史档案和管理效率。

主要使用 `/console`、任务页、档案页。

## 3. 核心用户故事

1. 作为项目评审，我希望在一个大屏页面看到项目解决什么问题、Agent 如何协作、能带来什么成效，以便快速判断项目价值。
2. 作为农技员，我希望选择一个地块并录入天气、土壤、苗情和病虫草害信息，以便发起智能分析。
3. 作为农技员，我希望系统展示多个 Agent 的分析过程，以便确认建议不是单点生成，而是经过完整推理。
4. 作为农场管理者，我希望系统自动生成农事任务，以便跟踪执行。
5. 作为项目管理人员，我希望每次分析都沉淀为档案，以便后续追溯、验收和复盘。

## 4. 信息架构

```text
/
├─ /demo                         宣传演示大屏

   ├─ /console/fields            地块管理
```

## 5. 功能需求

### 5.1 宣传演示大屏

#### 5.1.1 页面目标

展示项目价值、业务闭环和技术亮点，服务资金申请和路演展示。该页面不是普通官网首页，而是 16:9 深色数字农业指挥舱，视觉效果以用户提供的参考图为准。

#### 5.1.2 页面内容

- 顶部状态栏：
  - 天气、温度、湿度、风向/风力。
  - 标题：`AI驱动的大田作物稳产减灾决策平台`。
  - 作物范围：`水稻 | 小麦 | 玉米 | 大豆`。
  - 当前时间、系统状态、更多菜单。
- 左侧数据塔：
  - 综合态势：管理地块、覆盖面积、高风险地块、待办农事。
  - 作物种植分布环图。
  - 生育期分布堆叠条。
- 中央主视图：
  - 田间风险态势地图。
  - 水稻、小麦、玉米、大豆地块分区。
  - 风险热区、传感器点位、天气浮层、图层按钮、风险等级图例。
- 右侧决策区：
  - 多 Agent 决策链。
  - 风险预警卡片。
- 底部数据带：
  - 土壤墒情趋势折线图。
  - 农事任务队列。
  - 近期分析结果。

#### 5.1.3 验收标准

- 首屏能够看清项目名称、对象作物、核心指标和流程入口。
- 首屏整体视觉应接近用户提供的深色政府指挥中心参考图。
- 页面无需登录即可访问。
- 使用种子数据时页面不为空。
- 移动端和桌面端文字不重叠。
- 1920x1080 或 1440px 以上桌面视口下，大屏主体内容应完整、密集且有层次。
- 必须包含明显可见的动画和交互，不允许实现为静态截图式页面。

### 5.2 管理后台首页

#### 5.2.1 页面目标

提供农技员和管理者的工作入口。

#### 5.2.2 页面内容

- 顶部导航：大屏、工作台、地块、任务、档案。
- 核心指标：
  - 地块总数
  - 高风险地块
  - 待执行任务
  - 本周分析次数
- 高风险地块列表。
- 近期 Agent 报告。
- 今日待办农事。

#### 5.2.3 验收标准

- 能从首页进入地块列表、任务页和档案页。
- 高风险地块和待办任务数据与数据库一致。

### 5.3 地块管理

#### 5.3.1 字段

- 地块名称
- 位置
- 面积，单位亩
- 作物类型：水稻、小麦、玉米、大豆
- 品种
- 生育期
- 播种日期
- 目标产量，单位 kg/亩
- 风险等级

#### 5.3.2 功能

- 查看地块列表。
- 查看地块详情。
- MVP 阶段可先使用种子数据；新增/编辑作为增强项。
- 从地块详情进入数据录入和 Agent 分析。

#### 5.3.3 验收标准

- 种子数据包含水稻、小麦、玉米、大豆各一个地块。
- 地块详情展示最新观测、最新报告和相关任务。

### 5.4 数据录入

#### 5.4.1 输入项

天气：

- 当前温度
- 降雨量
- 风力
- 未来天气趋势

土壤：

- 土壤墒情
- 土壤温度
- pH
- 氮/磷/钾水平

苗情：

- 株高
- 叶色
- 长势等级

病虫草害：

- 虫害描述
- 病害描述
- 杂草描述

农事记录：

- 最近灌溉日期
- 最近施肥日期
- 最近用药日期
- 备注

#### 5.4.2 功能

- 提交观测数据。
- 保存后返回地块详情或进入 Agent 分析。
- 对关键数值做基础校验。

#### 5.4.3 验收标准

- 必填项缺失时提示用户。
- 数值字段不接受明显非法值，例如负面积、负温度以外的负墒情。
- 保存成功后可在地块详情看到最新观测。

### 5.5 Agent 决策中心

#### 5.5.1 页面目标

展示多 Agent 协作过程并触发分析。

#### 5.5.2 Agent 列表

1. 田间感知 Agent：整理地块、天气、土壤、苗情和农事数据。
2. 作物诊断 Agent：判断生育期、长势、缺水缺肥风险。
3. 病虫草害预警 Agent：识别病虫草害风险等级。
4. 水肥决策 Agent：生成灌溉、施肥、控旺、排水建议。
5. 灾害应对 Agent：针对干旱、暴雨、高温、低温、倒伏等风险给出处置建议。
6. 农事档案 Agent：沉淀诊断结果、建议、任务和追溯记录。

#### 5.5.3 长链推理流程

```text
识别作物与地块
→ 判断生育阶段
→ 融合天气、土壤、苗情、病虫草害和农事记录
→ 识别主要风险
→ 生成候选农事措施
→ 结合天气窗口和成本约束排序
→ 输出可执行建议
→ 生成任务
→ 进入档案
```

#### 5.5.4 验收标准

- 点击“开始分析”后生成 AgentRun。
- 每次分析包含 6 个 AgentStep。
- 生成风险等级、摘要、推理过程和建议列表。
- 分析完成后自动生成农事任务。

### 5.6 诊断报告

#### 5.6.1 报告内容

- 地块信息
- 作物信息
- 风险等级
- 综合判断
- 长链推理
- 农事建议
- Agent 步骤输出
- 自动生成的农事任务

#### 5.6.2 验收标准

- 报告可通过独立 URL 访问。
- 报告中至少包含 3 条农事建议。
- 报告内容应使用农业业务语言，避免泛化 AI 表述。

### 5.7 农事任务

#### 5.7.1 任务字段

- 标题
- 描述
- 地块
- 来源分析
- 优先级
- 截止日期
- 状态：待执行、已完成、已逾期

#### 5.7.2 功能

- 查看任务列表。
- 按状态分组。
- 标记任务为已完成。

#### 5.7.3 验收标准

- Agent 分析后至少生成一条任务。
- 任务状态变更后页面刷新仍保持状态。

### 5.8 农事档案

#### 5.8.1 页面内容

- 历史分析列表。
- 地块名称。
- 作物类型。
- 风险等级。
- 分析时间。
- 报告摘要。
- 报告详情入口。

#### 5.8.2 验收标准

- 每次 Agent 分析完成后自动进入档案。
- 档案记录可跳转到报告详情。

## 6. 非功能需求

### 6.1 可用性

- MVP 不依赖外部 AI Key 即可运行。
- 页面加载状态、空状态和错误状态清晰。
- 中文文本清楚、专业、适合农业项目申报场景。

### 6.2 性能

- 本地分析请求响应时间小于 10 秒。
- 首屏渲染不依赖大型地图或视频资源。

### 6.3 可维护性

- Agent 逻辑与 UI 解耦。
- 风险评分、Agent 编排、任务生成分别独立。
- 后续可替换 Mock Agent 为真实 LLM Agent。

### 6.4 安全与边界

- MVP 不提供医疗、金融或法律结论。
- 农事建议定位为辅助决策，不替代农技专家现场判断。
- 不执行真实设备控制。

### 6.5 大屏动效与交互

`/demo` 必须包含以下动效和交互：

- 指标数字计数动画。
- 地图高风险区域脉冲动画。
- 传感器点位信号波纹。
- Agent 节点顺序点亮。
- Agent 箭头流光。
- 环形图和折线图绘制动画。
- 高风险预警卡片边框脉冲。
- 近期分析列表自动滚动。
- 地块点击选择并高亮。
- 图层按钮激活状态切换。
- 风险预警点击联动地图地块。
- Agent 节点 hover 展示摘要。
- 图表 hover tooltip。

动效必须支持 `prefers-reduced-motion`，减少动画设置下应降低循环动画强度。

## 7. 数据种子

MVP 需内置 4 个示范地块：

1. 江淮水稻示范田：水稻，分蘖期。
2. 黄淮麦田示范区：小麦，灌浆期。
3. 东北玉米示范田：玉米，拔节期。
4. 黑土地大豆示范田：大豆，开花期。

## 8. 演示脚本

1. 打开 `/demo`，介绍项目痛点和四类作物。
2. 展示多 Agent 协作流程。
3. 进入 `/console`，查看高风险地块和待办任务。
4. 进入玉米示范田，录入高温少雨、墒情偏低、叶片卷曲等信息。
5. 发起 Agent 分析。
6. 查看长链推理报告。
7. 查看自动生成的灌溉、巡田和复查任务。
8. 打开档案页，说明报告可追溯、可验收。
