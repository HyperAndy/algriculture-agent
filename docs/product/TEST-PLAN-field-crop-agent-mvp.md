# 大田作物稳产减灾 Agent 测试计划

## 1. 文档目的

本文档定义 MVP 阶段的测试范围、测试类型、关键用例和验收标准，确保系统在进入演示和资金申请展示前具备稳定可运行的闭环能力。

## 2. 测试目标

- 验证核心业务流程可跑通。
- 验证风险评分规则符合预期。
- 验证多 Agent 分析输出结构稳定。
- 验证报告、任务、档案自动生成。
- 验证宣传大屏和管理后台页面可访问、可理解、无明显布局问题。

## 3. 测试范围

包含：

- 数据模型和种子数据。
- 风险评分。
- Mock Agent 编排。
- 农事任务生成。
- API 路由。
- 管理后台核心页面。
- 宣传演示大屏。
- 端到端演示流程。

不包含：

- 真实遥感识别准确率。
- 真实 IoT 设备接入。
- 真实大模型效果评估。
- 用户权限和安全渗透测试。

## 4. 单元测试用例

### 4.1 风险评分

| 用例 | 输入 | 预期 |
|---|---|---|
| 高温低墒情 | 墒情 38%，温度 34℃，降雨 0mm | 干旱高风险 |
| 正常墒情 | 墒情 65%，温度 26℃，降雨 8mm | 干旱低风险 |
| 病虫描述 | 虫害描述“少量虫咬叶片” | 生物中风险 |
| 严重病害 | 病害描述“病斑扩散，成片发生” | 生物高风险 |
| 多风险合并 | 干旱 high，病虫 medium | 综合 high |

### 4.2 Agent 编排

| 用例 | 操作 | 预期 |
|---|---|---|
| 标准分析 | 对玉米示范田发起分析 | 生成 6 个 AgentStep |
| 报告摘要 | 分析水稻示范田 | 摘要包含“水稻” |
| 建议数量 | 任意分析 | 至少 3 条建议 |
| 风险枚举 | 任意分析 | 风险值属于 low/medium/high |

### 4.3 任务生成

| 用例 | 输入 | 预期 |
|---|---|---|
| 干旱高风险 | 墒情 38%，高温少雨 | 生成“安排补灌” |
| 病虫害中风险 | 虫害描述不为空 | 生成“开展病虫草害复查” |
| 养分偏低 | 钾素 low | 生成“评估追肥方案” |
| 任意分析 | 完成 AgentRun | 生成“复查地块状态” |

## 5. API 测试用例

### 5.1 地块 API

- `GET /api/fields` 返回 200。
- 返回列表包含 4 个示范地块。
- 每个地块包含 `id`、`name`、`cropType`、`growthStage`、`riskLevel`。

### 5.2 观测 API

- `POST /api/fields/[id]/observations` 输入合法数据返回 201。
- 缺少必填字段返回 400。
- `soilMoisturePercent < 0` 或 `> 100` 返回 400。

### 5.3 分析 API

- 有观测数据时，`POST /api/fields/[id]/analyze` 返回 200。
- 无观测数据时返回 400，并提示先录入数据。
- 分析成功后生成 AgentRun、AgentStep 和 FarmingTask。

### 5.4 报告 API

- `GET /api/reports/[id]` 返回报告详情。
- 不存在的报告 ID 返回 404。

### 5.5 任务 API

- `GET /api/tasks` 返回任务列表。
- `PATCH /api/tasks` 可将任务标记为 completed。
- 不存在的任务 ID 返回 404。

## 6. 页面验收用例

### 6.1 `/demo`

验收：

- 首屏展示项目名称：`AI驱动的大田作物稳产减灾决策平台`。
- 整体视觉为深色政府/项目申报指挥中心风，接近用户提供的参考图。
- 展示水稻、小麦、玉米、大豆。
- 展示顶部天气、湿度、风向、时间、系统状态。
- 展示左侧综合态势、作物种植分布、生育期分布。
- 展示中央田间风险态势地图，且地图是最大视觉核心。
- 展示右侧多 Agent 决策链和风险预警。
- 展示底部土壤墒情趋势、农事任务队列、近期分析结果。
- 页面在 1440px 及以上桌面宽度无明显文字重叠。
- 中文文本无乱码。

动效验收：

- 指标数字计数动画可见。
- 高风险地块脉冲动画可见。
- 传感器点位信号波纹可见。
- Agent 节点顺序点亮可见。
- Agent 箭头流光可见。
- 环形图绘制动画可见。
- 折线图绘制动画可见。
- 高风险预警边框脉冲可见。
- 近期分析自动滚动可见。

交互验收：

- 地块点击后可选中并高亮。
- 图层按钮点击后可切换激活状态。
- 风险预警点击后可联动高亮对应地块。

## 7. 端到端演示用例

### 用例：玉米拔节期干旱风险分析

步骤：

1. 运行数据库 seed。
2. 打开 `/console/fields`。
3. 选择“东北玉米示范田”。
4. 进入数据录入页。
5. 输入：
   - 温度 34℃
   - 降雨 0mm
   - 未来 5 天高温少雨
   - 墒情 38%
   - 钾素 low
   - 叶色 pale
   - 长势 weak
   - 虫害描述“少量虫咬叶片”
6. 提交观测。
7. 发起 Agent 分析。
8. 查看报告。
9. 查看任务列表。
10. 查看档案。

预期：

- 综合风险为 high。
- 报告提到干旱或高温少雨。
- 生成补灌任务。
- 生成病虫草害复查任务。
- 档案页出现本次分析记录。

## 8. 异常用例

| 场景 | 操作 | 预期 |
|---|---|---|
| 无观测数据分析 | 直接点击分析 | 提示先录入数据 |
| 非法墒情 | 输入 120% | 表单或 API 拒绝 |
| 不存在地块 | 访问错误 ID | 404 或友好错误 |
| 不存在报告 | 访问错误报告 ID | 404 |
| 任务重复完成 | 已完成任务再次完成 | 状态保持 completed |
