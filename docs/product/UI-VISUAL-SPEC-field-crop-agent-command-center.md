# 大田作物稳产减灾决策平台前端视觉与动效规格

## 1. 文档目的

本文档锁定最终前端视觉方向：`/demo` 必须实现为政府/项目申报指挥中心风格的大屏，整体效果以用户提供的参考图为准。实现时不能再采用普通官网式、白底卡片式或轻量后台式展示。

本文档同时定义大屏布局、视觉系统、图表模块、交互行为、动效要求和验收标准，确保最终页面具备专业大盘报表展示质感，而不是静态信息拼接。

## 2. 最终风格定位

最终实现风格：

**政府/项目申报指挥中心风大屏**

关键词：

- 深色数字农业指挥舱
- 中央田间风险态势地图
- 左右数据塔
- 多 Agent 决策链
- 风险预警面板
- 农事任务队列
- 土壤墒情趋势
- 近期分析动态
- 有节制的科技光效
- 高信息密度但层次清楚

不得采用：

- 普通浅色 SaaS 首页作为 `/demo`
- 纯营销落地页
- 白底大卡片堆叠
- 只靠静态文字说明项目
- 只有图表没有农业场景主视觉
- 过度赛博、霓虹、炫光导致文字不可读

## 3. 大屏画布

### 3.1 画布尺寸

主目标尺寸为 `1920x1080`，按 16:9 大屏设计。

页面在普通浏览器中可滚动或自适应，但首屏应尽量完整容纳主要大屏内容。桌面宽屏优先级高于移动端。

### 3.2 背景

背景应为深色数字农业底图：

- 基础色：深蓝黑、墨绿黑。
- 可使用细网格、暗纹、扫描线、地图纹理、轻微噪声。
- 不使用大面积渐变球、装饰光斑或无意义漂浮元素。
- 背景不能抢过数据内容。

### 3.3 整体布局

采用“三列 + 顶部 + 底部”的大屏结构：

```text
┌──────────────────────────────────────────────────────────────┐
│ 顶部状态栏：天气 / 标题 / 作物范围 / 时间 / 系统状态            │
├───────────────┬──────────────────────────────┬───────────────┤
│ 左侧数据塔     │ 中央田间风险态势地图           │ 右侧Agent与预警 │
│ 综合态势       │ 地块分区 / 风险热区 / 图层控件  │ 多Agent决策链   │
│ 作物种植分布   │ 气象/传感器浮层                │ 风险预警        │
│ 生育期分布     │ 比例尺 / 图例                  │                │
├───────────────┴──────────────┬───────────────┴───────────────┤
│ 土壤墒情趋势                  │ 农事任务队列 / 近期分析结果     │
└──────────────────────────────────────────────────────────────┘
```

## 4. 页面模块规格

### 4.1 顶部状态栏

必须包含：

- 左侧天气组：温度、天气、湿度、风向/风力。
- 中央主标题：`AI驱动的大田作物稳产减灾决策平台`。
- 标题下方作物范围：`水稻 | 小麦 | 玉米 | 大豆`。
- 右侧时间：示例 `2025-05-20 10:25:36`。
- 右侧系统状态：`系统正常`，带绿色状态点。
- 右侧更多/菜单图标。

动效：

- 系统状态绿点应有轻微呼吸动画。
- 顶部标题可有极轻微入场淡入，不做持续闪烁。

### 4.2 左侧综合态势

模块标题：`综合态势`

指标：

- 管理地块：`4 块`
- 覆盖面积：`710 亩`
- 高风险地块：`1 块`
- 待办农事：`4 项`

视觉：

- 每个指标为横向深色信息条。
- 左侧使用 lucide 图标或同风格线性图标。
- 右侧大数字使用高亮色。
- 高风险数字使用红橙色。

动效：

- 数字首次加载时使用 600-900ms 计数动画。
- 指标条入场时从左到右轻微滑入。
- hover 时边框亮度提升，不能导致布局位移。

### 4.3 作物种植分布

模块标题：`作物种植分布（亩）`

内容：

- 环形图，中心显示 `总面积 710亩`。
- 图例：
  - 水稻 210（29.6%）
  - 小麦 180（25.4%）
  - 玉米 200（28.2%）
  - 大豆 120（16.8%）

动效：

- 环形图初次加载时从 0 到目标角度绘制。
- hover 某一作物时高亮对应图例和扇区。

### 4.4 生育期分布

模块标题：`生育期分布`

内容：

- 每个作物一条堆叠进度条。
- 阶段包括：播种期、苗期、拔节期、孕穗期、灌浆期、成熟期。
- 进度条应使用绿色、黄绿色、黄色、蓝色等分段色。

动效：

- 进度条加载时从左到右填充。
- hover 显示阶段名称和百分比。

### 4.5 中央田间风险态势地图

模块标题：`田间风险态势地图`

这是页面视觉核心，必须最大、最醒目。

内容：

- 使用航空农田感背景或代码绘制的农田网格场景。
- 叠加多个地块边界，多边形或块状区域。
- 地块标签：
  - 水稻
  - 小麦
  - 玉米
  - 大豆
- 风险颜色：
  - 安全/低风险：绿色
  - 中风险：黄色/橙色
  - 高风险：红色
- 至少一个红色高风险地块，带警告图标。
- 传感器点位：气象站、墒情站、虫情站等小图标。
- 地图左上角悬浮天气卡片：温度、湿度、风速、降雨量。
- 右侧图层控制按钮：图层、气象、土壤、病虫、灾害、实景。
- 右上角风险等级图例。
- 左下角比例尺。

动效：

- 高风险地块必须有红色脉冲/呼吸光圈。
- 传感器点位应有微弱信号波纹动画。
- 地图边界线可有慢速描边或淡光流动。
- 图层按钮 hover/点击时应切换激活状态。
- 点击地块时，该地块高亮，并更新中央/底部的选中地块摘要。
- 地图初始加载时从暗到亮渐显，避免突兀。

### 4.6 右侧多 Agent 决策链

模块标题：`多Agent决策链`

内容：

以 2x3 网格或流程链形式展示：

- 田间感知：多源数据采集
- 作物诊断：长链推理分析
- 风险预警：地块级预警
- 水肥决策：精准水肥建议
- 灾害应对：应急处置方案
- 农事档案：全过程追溯

视觉：

- 每个 Agent 是图标节点。
- 节点之间用箭头连接。
- 图标采用绿色发光线性风格。
- 当前运行或最近运行节点应高亮。

动效：

- 节点按顺序点亮，形成“Agent 正在协同”的视觉感。
- 箭头上有轻微流光动画。
- hover 节点显示该 Agent 的输出摘要。
- 点击节点可在旁边或弹层展示输入/输出摘要。

### 4.7 风险预警

模块标题：`风险预警`

至少展示两条：

- `玉米拔节期干旱风险`，高风险。
- `小麦灌浆期干热风风险`，中风险。

每条预警包含：

- 风险图标。
- 标题。
- 风险等级标签。
- 涉及地块。
- 预警时间。
- 影响面积。
- `查看详情`按钮。

动效：

- 高风险卡片应有较轻的红色边框脉冲。
- 新预警进入时使用上滑淡入。
- 点击 `查看详情` 可高亮中央地图对应地块。

### 4.8 底部土壤墒情趋势

模块标题：`土壤墒情趋势`

内容：

- 折线图，展示 05-14 到 05-20 趋势。
- 至少 4 条线：1号地块小麦、2号地块水稻、3号地块玉米、4号地块大豆。
- Y 轴为土壤含水率百分比。

动效：

- 折线首次加载时从左到右绘制。
- hover 某个日期点时显示 tooltip。
- 线条和图例联动高亮。

### 4.9 农事任务队列

模块标题：`农事任务队列`

表格列：

- 任务类型
- 任务描述
- 涉及地块
- 建议时间
- 优先级
- 状态

示例任务：

- 灌溉：3号地块玉米灌溉建议。
- 追肥：1号地块小麦追肥建议。
- 病虫防治：2号地块稻飞虱防治。
- 除草：4号地块大豆除草建议。

动效：

- 表格行 hover 高亮。
- 状态 `待执行` 可有轻微闪烁或绿色呼吸点。
- 高优先级标签使用红色，中优先级橙色，低优先级绿色。

### 4.10 近期分析结果

模块标题：`近期分析结果`

内容：

- 按时间倒序展示 Agent 分析摘要。
- 每条包含图标、摘要、时间。
- 示例：
  - `基于气象预测，未来3天玉米干旱风险持续，请及时灌溉`
  - `1号地块小麦灌浆速度偏慢，建议叶面补充磷钾肥`
  - `2号地块稻飞虱发生风险上升，建议加强监测与防治`

动效：

- 列表可轻微自动滚动。
- 新项出现时淡入。
- hover 时停止滚动，便于阅读。

## 5. 视觉系统

### 5.1 色彩

建议色板：

- 背景深色：`#041316`、`#071C22`、`#09242B`
- 面板底色：`rgba(6, 34, 40, 0.78)`
- 面板边框：`#1E8F80`、`rgba(110, 255, 194, 0.35)`
- 主高亮绿：`#7EE787`
- 科技青：`#33D6FF`
- 麦穗黄：`#F5C84C`
- 风险橙：`#FF9F1C`
- 高风险红：`#FF5A4F`
- 正文浅色：`#D8F3E8`
- 次级文字：`#88A8A3`

### 5.2 字体

- 中文优先：`Microsoft YaHei`、`PingFang SC`、`Noto Sans SC`。
- 数字可使用更硬朗的等宽或半等宽风格。
- 标题必须大、稳、清晰，不能使用花体。

### 5.3 面板

- 面板为深色透明玻璃态，但透明度不能影响可读性。
- 面板边框可带角标、细线、轻微发光。
- 圆角控制在 6-8px。
- 同一页面不能出现白底卡片。

### 5.4 图标

优先使用 `lucide-react` 图标：

- 天气：`CloudSun`
- 湿度：`Droplets`
- 风：`Wind`
- 地块：`Map`
- 告警：`TriangleAlert`
- 任务：`ClipboardList`
- Agent：`Sprout`、`Leaf`、`Wheat`、`CloudRain`、`FileText`
- 系统状态：`CircleCheck`

## 6. 交互要求

必须实现：

- 地块点击选择。
- 图层按钮激活状态切换。
- 风险预警点击联动地图地块高亮。
- Agent 节点 hover 显示摘要。
- 任务行 hover 高亮。
- 图表 hover tooltip。
- 近期分析列表 hover 暂停自动滚动。

可选增强：

- 全屏按钮。
- 自动轮播地块。
- 一键进入管理后台。
- 点击任务跳转到任务详情或后台任务页。

## 7. 动效要求

动效是本次大屏改造的核心要求之一，必须实现，不允许只做静态页面。

必须包含：

- 页面模块入场动画：淡入 + 轻微位移。
- 指标数字计数动画。
- 地图高风险区域脉冲动画。
- 传感器点位信号波纹。
- Agent 决策链节点顺序点亮。
- Agent 箭头流光。
- 折线图绘制动画。
- 环形图绘制动画。
- 预警卡片高风险边框脉冲。
- 近期分析结果轻微自动滚动。

动效原则：

- 动效要服务“系统正在运行”的感知。
- 动效节奏克制，不能影响阅读。
- 支持 `prefers-reduced-motion`：用户开启减少动画时，应降低或关闭循环动画。
- 不允许大量无意义闪烁。
- 不允许动效导致布局抖动。

## 8. 数据映射

MVP 可使用现有 seed 数据和推导数据：

- 管理地块：`fields.length`
- 覆盖面积：`sum(fields.areaMu)`
- 高风险地块：`fields.filter(riskLevel === "high").length`
- 待办农事：`pendingTasks.length`
- 作物面积结构：按 `cropType` 汇总 `areaMu`
- 风险预警：来自高/中风险地块、最新 AgentRun、固定示范预警
- Agent 决策链：来自 `AgentStep`
- 农事任务队列：来自 `FarmingTask`
- 近期分析结果：来自 `AgentRun`

地图背景和地块边界 MVP 可用代码绘制或 CSS/SVG 模拟，不要求真实 GIS。

## 9. 验收标准

`/demo` 必须满足：

- 首屏视觉接近用户提供的参考图，而不是当前浅色卡片页。
- 顶部标题、天气、时间、系统状态完整。
- 左侧综合态势、作物分布、生育期分布完整。
- 中央田间风险态势地图为页面最大视觉核心。
- 右侧多 Agent 决策链和风险预警完整。
- 底部土壤墒情趋势、农事任务队列、近期分析结果完整。
- 至少 8 类动效实际可见。
- 至少 5 类交互可操作。
- 中文文本无乱码。
- `npm run build` 通过。
- 桌面 1440px 及以上无明显重叠。
- 1920x1080 视口下达到大屏展示效果。
