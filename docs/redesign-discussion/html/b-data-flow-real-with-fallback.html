/demo Big Screen

只消费 ViewModel，不直接访问 Prisma，不直接拼接业务规则。这样视觉组件保持纯展示。

  • 稳定视觉结构
  • 真实数据标识
  • 演示兜底标识
规则 1：真实数据完整

使用真实后台聚合数据；大屏角落可显示“实时业务数据”。适合内部试用和产品演示。

规则 2：真实数据部分缺失

真实数据优先，缺失图表或模块用演示数据补齐；模块级标记来源，保证大屏不空。

规则 3：真实数据异常

回退到完整演示 ViewModel，同时保留错误日志。路演时页面不能崩，也不能出现半屏空白。

设计边界说明
大屏组件纯展示组件，只接收 `CommandDashboardViewModel`。
数据聚合放在 `features/dashboard/server`，负责查询真实数据和 fallback 合并。
演示数据放在 `features/dashboard/demo-data`，作为确定性 seed，不散落在组件里。
测试重点覆盖三种场景：真实完整、真实缺失、真实异常回退。
