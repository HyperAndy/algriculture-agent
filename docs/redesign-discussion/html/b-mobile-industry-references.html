移动端样式参考：企业级农业 SaaS

参考 Material Design、Ant Design、SaaS Dashboard 和企业移动端实践，形成适合本项目的移动端交互规范。

Material Design

底部导航 + Bottom Sheet

  • 底部导航只放 3-5 个一级高频入口。
  • 筛选、更多操作、批量动作放 Bottom Sheet。
  • 触控目标不小于 44px，避免误触。
Ant Design

清晰导航 + 分步骤任务

  • 减少页面跳转，保证用户知道当前位置。
  • 复杂录入用 Steps 拆成明确流程。
  • 列表、表单、反馈组件保持一致。
Enterprise UX

任务优先，不追求娱乐化

  • 用户目标是完成工作，不是探索页面。
  • 优先展示待处理事项和下一步动作。
  • 弱化装饰，强化状态、反馈、错误恢复。

工作台地块任务报告
1. 首页不是仪表墙

移动端只展示关键 KPI、风险事项、今日任务和主操作入口，避免桌面端指标全部塞入。

2. 表格卡片化

地块、任务、报告列表在移动端变成摘要卡；详细字段进入详情页或展开区域。

3. 录入分步骤

天气、土壤、苗情、病虫草害、农事记录分步完成，支持草稿保存。

4. 操作固定可达

关键提交、启动分析、保存任务等按钮固定在底部或当前卡片底部。

5. 筛选用 Sheet

移动端筛选和批量操作用底部 Sheet，不占据主列表空间。

6. 状态反馈明确

长任务显示运行中、失败重试、上次同步时间；不出现无解释空白。
