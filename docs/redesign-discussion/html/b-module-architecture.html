B 方案：模块化单体 + 双前端体验

核心原则：业务内核共享，前端体验隔离。大屏是“演示驾驶舱”，后台是“业务工作台”，两者不再混用样式和组件。

1. Frontend Experience Layer

Big Screen App

`/demo`，科幻农业指挥中心。使用独立设计令牌、独立大屏组件、独立动效系统。

Console App

`/console`，企业级农业 SaaS 管理后台。强调表单、列表、详情、任务流和报告可读性。

Shared UI Primitives

按钮、Badge、Tooltip、Dialog、Table 等基础组件可共享，但主题变量分开。

2. Feature Modules

fields

地块、作物、品种、面积、生育期、风险级别。

observations

天气、土壤、苗情、病虫草害、农事记录录入。

agent-runs

Agent 编排、步骤记录、推理摘要、建议生成。

tasks

农事任务生成、状态流转、优先级、到期提醒。

reports

诊断报告、归档、导出、可追溯结果。

dashboard

给 `/demo` 和 `/console` 提供不同 ViewModel，而不是组件里拼数据。

3. Application / Domain / Infrastructure

Application Services

面向用例：创建观测、运行分析、生成任务、获取驾驶舱数据。

→
Domain Core

风险规则、作物阶段、Agent 输出契约、任务生成规则。

→
Infrastructure

Prisma Repository、SQLite/Postgres、Mock/LLM Agent Provider、外部数据适配器。

重构重点：把现在散落在页面、API route、组件里的业务判断，收敛到 `features/*` 和 `domain/*`。
